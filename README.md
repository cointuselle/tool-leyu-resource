# tool-leyu-resource

本仓库用于归档和发布多个独立 HTML 页面，主要面向静态资源展示与分发场景。所有页面均为独立打包的 HTML 文件，不依赖特定服务端环境，可直接在浏览器中打开运行。

## 项目简介

tool-leyu-resource 是一个通用 HTML 页面资源仓库，旨在集中管理并发布各类独立功能的 HTML 页面。仓库内容不针对任何特定域名或网站，页面之间彼此独立，可单独使用或组合部署。

## 目录说明

```
.
├── README.md          # 本文件
├── pages/             # 存放所有 HTML 页面文件
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
└── assets/            # 页面引用的公共资源（图片、样式、脚本等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：所有发布的 HTML 页面均放置在此目录下，文件名应保持唯一且具备一定可读性。
- `assets/`：存放页面共用的静态资源，按类型分目录组织。若页面使用独立资源，建议在页面同级目录下建立 `assets` 文件夹。

## 页面归档说明

- 每个 HTML 页面应自包含其核心功能，外部依赖需明确声明并放置于 `assets/` 目录。
- 页面应避免硬编码任何特定网络地址或服务端点，以保持通用性与可移植性。
- 归档时请确保页面在主流浏览器（Chrome、Firefox、Edge、Safari）中可正常渲染与交互。
- 建议为每个页面添加简要注释说明其用途与使用方式。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有公共资源需求同时更新 `assets/`。
- 更新页面：直接替换 `pages/` 中对应文件，并同步更新相关资源。
- 删除页面：移除 `pages/` 中对应文件，并清理不再被引用的资源文件。
- 提交前请检查文件完整性及链接有效性，避免引入损坏或无效引用。

## 贡献与使用

本仓库欢迎对现有页面的改进或新页面的提交。提交前请确保内容符合通用性与独立性原则，不包含特定商业或营销信息。使用者可自由下载、部署或修改仓库内的页面，但需自行承担相关责任。
